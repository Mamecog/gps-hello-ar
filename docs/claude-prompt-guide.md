# Claude Code への依頼プロンプトガイド

## プロジェクト説明テンプレート（毎回渡すと精度が上がる）

```
このプロジェクトはブラウザ完結の顔検出WebARアプリです。
- index.html 一ファイルにHTML/CSS/JSが集約されています
- face-api.js（CDN）+ TinyFaceDetectorモデル（./models/）で顔検出
- getUserMedia でインカメラを取得し、<canvas>にハートを描画
- detectLoop（15fps）と renderLoop（60fps）の2ループ構成
- デプロイ先: GitHub Pages（HTTPS）
```

---

## 修正依頼の例

### 見た目の変更
```
index.html を読んでください。
renderLoop() 内で顔の上に固定表示している ❤️ を ⭐ に変えてください。
spawnHearts() のパーティクルも同じ絵文字にしてください。
```

### パーティクル調整
```
index.html を読んでください。
spawnHearts() と renderLoop() を確認して、以下を変更してください：
- パーティクルの生成数を 1回に 5〜8個 に増やす
- 浮かぶスピードを現在の 1.5倍 にする
```

### カメラ切り替え
```
index.html を読んでください。
getUserMedia で facingMode: 'user'（インカメ）を使っています。
リアカメラ（'environment'）に切り替えてください。
リアカメラでは鏡像反転（scaleX(-1)）が不要なのでCSSも合わせて修正してください。
```


## 依頼時の注意事項

**「index.html を読んでください」を最初に書く**
Claude はファイルを読まずに推測で回答することがある。
明示的に読むよう指示すると根拠ベースの回答になる。

**変更範囲を限定して指示する**
「renderLoop() の中だけ変更して」のように範囲を絞ると意図しない変更が減る。

**動作確認条件を伝える**
「iOS の Safari で動作確認が必要」「HTTPS 環境でしか動かない」など制約を最初に伝える。

**「動作を変えずに」と明示する**
リファクタリングや整理の場合は「既存の動作を変えないで」と書くと安全。
